# How to Use This Guide

<span class="label-time">5 minutes</span>
<span class="label-difficulty">Beginner</span>

## When should I read this?

Read this page before starting the guide. It explains how pages are organized and how to move between foundational instructions, advanced methods, and complete pipeline examples.

## What each page contains

Most pages follow the same structure:

1. When to read the page
2. What you will learn
3. Why the topic matters
4. Step-by-step implementation
5. Expected output
6. Common pitfalls
7. Performance considerations
8. Validation checklist
9. Reusable template
10. Related pages

## How to move through the guide

Beginners should start with **Getting Started**, then proceed to **Working with EHR Data**. Experienced analysts can use the search bar or sidebar to jump directly to a specific task.

{: .wisdom }
Do not copy code until you understand the expected input, the participant unit of analysis, and the validation output.
