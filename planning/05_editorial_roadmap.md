# 1. Purpose

This roadmap serves as the master editorial plan for the Field Guide. It tracks every planned section, page, and complete pipeline, providing a single source of truth for development priorities.

This document should be updated throughout the life of the project.

---

# 2. Development Philosophy

Content should be published in layers.

1. Build foundational concepts.
2. Build reusable methods.
3. Build domain-specific workflows.
4. Build complete end-to-end pipelines.
5. Expand with advanced topics and reference material.

No pipeline should introduce concepts that have not first been explained in a reusable guide.
