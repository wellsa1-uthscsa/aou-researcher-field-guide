# Contributor Guide

**Version:** 1.0.0  
**Status:** Complete  
**Effective Date:** July 17, 2026  
**Document role:** Internal project specification  
**Visibility:** Repository only

---

# Purpose

This guide establishes standards for contributing content to the Field Guide.

The primary goals are:

- Maintain scientific accuracy.
- Preserve editorial consistency.
- Promote reproducibility.
- Ensure long-term maintainability.

---

# Core Principles

Contributors should:

- Teach scientific reasoning before software.
- Explain why before how.
- Use reusable methods.
- Write for graduate-level learners.
- Follow the Editorial Style Guide.
- Use the Standard Page Template.

---

# Workflow

1. Plan the page within the Editorial Roadmap.
2. Create a development branch.
3. Draft the content.
4. Verify code examples.
5. Confirm citations.
6. Review formatting.
7. Submit for editorial review.
8. Merge after approval.

---

# Content Standards

Every page should:

- Address one primary question.
- Include learning objectives.
- Explain scientific motivation.
- Contain validated code.
- Include references.
- Link to related guides.

---

# Figures

Preferred:

- SVG
- Mermaid
- Annotated screenshots

Every figure requires:

- Caption
- Alt text
- Source attribution when appropriate

---

# Code

Code should be:

- Tested
- Commented
- Reproducible
- Minimal

Avoid unnecessary complexity.

---

# Citations

Prefer:

1. Primary literature
2. Official documentation
3. Authoritative review articles

---

# Review Checklist

Before merging:

- Scientific content verified
- Code executed successfully
- Figures reviewed
- Formatting consistent
- Links functional
- Metadata complete

---

# Repository Etiquette

- Keep commits focused.
- Use descriptive commit messages.
- Avoid unrelated formatting changes.
- Preserve stable URLs.
- Update the changelog for major additions.

---

# Future Contributors

As the project grows, contributors should view themselves as educators rather than authors. The objective is to build a cohesive educational resource, not a collection of independent tutorials.

---

# Completion Standard

A contribution is complete when it integrates seamlessly into the Field Guide and is indistinguishable in quality, structure, and tone from existing content.
